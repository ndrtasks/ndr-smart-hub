# NDR Smart Hub — Phase 5 Quality Pass

تمت إعادة ضبط منطق مذكرة الحضور HR-F-25 واختباره كسيناريوهات عمل وليس كحقول ثابتة.

## ما تم تصحيحه
- فصل محادثة NDR AI لكل مستخدم في التخزين المحلي.
- تمييز الاستئذان المستقبلي عن تصحيح حالة حضور حدثت فعليا.
- مذكرة الحضور أصبحت ديناميكية حسب نوع الحالة.
- الغياب لا يطلب أوقات دخول وخروج، ويطلب نوع العذر والمستند فقط عند اختياره.
- التأخر يطلب وقت الدخول فقط.
- الخروج المبكر يطلب وقت الخروج فقط.
- البصمة المفقودة تحدد دخول/خروج/كلاهما ثم تظهر الأوقات اللازمة فقط.
- العمل عن بعد: يوم كامل لا يطلب أوقات، وجزء من اليوم يطلب الدخول والخروج.
- عرض الوقت في المعاينة بصيغة واضحة ص/م بدلا من شرطات فارغة.
- معاينة النموذج لا تعرض الحقول غير المطلوبة للحالة.
- مسار HR-F-25 الافتراضي: المدير المباشر ثم المختص المسؤول في الموارد البشرية.
- لموظف شؤون المتدربين التجريبي TA001: TAM01 ثم E001 (نادر).
- نادر لا يستطيع اعتماد طلبه بنفسه؛ يتم اختيار بديل تلقائي في مرحلة HR.
- إشعار المرحلة يصل للمسؤول الحالي فقط.
- مدير القسم يعيد الطلب لمقدم الطلب فقط.
- الموارد البشرية تستطيع الرجوع إلى الموظف أو مرحلة سابقة حسب الصلاحية.

## اختبارات آلية منفذة
- بستأذن ساعتين -> إذن قصير
- تأخرت -> مذكرة حضور / متأخر
- نسيت بصمة الخروج -> مذكرة حضور / بصمة خروج
- غياب -> بدون حقول أوقات
- خروج مبكر -> وقت خروج فقط
- عمل عن بعد يوم كامل/جزء من اليوم
- المرفق الشرطي للغياب
- TA001 -> TAM01 -> E001
- إشعار المدير أولا ثم HR فقط
- تقييد خيارات الإعادة لمدير القسم
- عزل مفاتيح المحادثة بين E001 وTA001
